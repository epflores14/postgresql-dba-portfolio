# Project 3: Performance Monitoring & Query Optimization

## 📌 Objective
To analyze query performance in PostgreSQL, identify bottlenecks, and apply optimization techniques such as indexing and query restructuring. This project demonstrates how database administrators improve efficiency and reduce execution time.

---
## ⚙️ Steps Taken
1. **Baseline Query Execution**
   - Ran initial queries using `EXPLAIN ANALYZE`.
   - Captured execution times and query plans.

2. **Index Creation**
   - Added indexes on frequently queried columns.
   - Re-ran queries to measure performance improvements.

3. **Query Optimization**
   - Rewrote aggregate queries for efficiency.
   - Compared before-and-after execution plans.
 

4. **Performance Comparison**
   - Documented execution times before and after optimization.
   - Example:
     - Before: `1200 ms`
     - After: `150 ms`

---

## 📊 Results
- Queries now run significantly faster, improving overall database responsiveness.
- Demonstrates practical DBA skills in monitoring and tuning PostgreSQL workloads.
